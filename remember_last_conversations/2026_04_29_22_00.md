## 1. Correção Crítica — Dashboard `__next_error__`

### Problema
O dashboard falhava ao carregar com `<html id="__next_error__">` na primeira visita. A causa raiz era que após o SQL cleanup (DELETE de dados transacionais), os componentes de gráfico recebiam arrays vazios e executavam operações matemáticas inválidas:

- `LineChartNative`: `i / (data.length - 1)` → divisão por zero quando `length = 0` ou `NaN` quando `length = 1`
- `DonutChartNative`: `item.value / total * 100` → divisão por zero quando `total = 0`
- `dashboard/page.tsx`: `data = null` após erro de query → acesso a `data.kpis.total_tickets` crashava o React

### Arquivos Modificados
| Arquivo | Mudança |
|---------|---------|
| `src/components/dashboard/charts/LineChartNative.tsx` | Guard para array vazio; função `getX()` segura para `length === 1` |
| `src/components/dashboard/charts/DonutChartNative.tsx` | Guard para array vazio ou `total === 0` |
| `src/app/dashboard/page.tsx` | `EMPTY_DATA` fallback; `hasError` state; `display` alias seguro |

### Solução Técnica

**LineChartNative:**
```tsx
if (!data || data.length === 0) return <div>Sem dados no período</div>;
const getX = (i: number) => data.length === 1 ? chartWidth / 2 : (i / (data.length - 1)) * ...;
```

**DonutChartNative:**
```tsx
if (!data || data.length === 0 || total === 0) return <div>Sem dados</div>;
```

**dashboard/page.tsx:**
```tsx
const EMPTY_DATA = { kpis: {...zeros}, savings_timeline: [], ... };
const [hasError, setHasError] = useState(false);
// em catch: setHasError(true); setData(EMPTY_DATA);
const display = data ?? EMPTY_DATA;
```

- Dashboard exibe painel zerado (KPIs = 0, gráficos com mensagem "Sem dados") em vez de crashar
- Banner de aviso mostrado quando dados falham ao carregar
